PHP + Nginx + PostgreSQL

Raspberry Pi Home Server Hack #6 >> As a pre-requisite to host web applications like Tiny Tiny RSS on my Raspberry Pi I install PHP, the lightweight web server Nginx, and the PostgreSQL database.

0. PHP

Install:

$ sudo apt-get update
$ sudo apt-get install php5 php5-fpm php-apc php5-curl php5-cli php5-pgsql php5-gd php5-mcrypt

To improve security [1] edit /etc/php5/fpm/php.ini and change pathinfo to 0:

cgi.fix_pathinfo=0

Restart PHP:

$ sudo /etc/init.d/php5-fpm restart

1. Nginx

Install:

$ sudo apt-get install nginx
$ sudo /etc/init.d/nginx start

To verify that the web server is running, open a browser and navigate to http://your.raspberry.pi.ip. If you see Welcome to Nginx the server is installed correctly. Nginx is capable of serving up many different web domains or server blocks (virtual hosts) from the same server:

  • block content is placed in subfolders located in /usr/share/nginx/www
  • configuration in /etc/nginx/sites-available
  • a server block is made active by setting a symbolic link in /etc/nginx/sites-enabled to its config file

2. PostgreSQL

Install:

$ sudo apt-get postgresql

Login and change password for the postgresql user:

$ sudo -u postgres psql postgres
\password postgres  # hit enter and enter password

PostgreSQL maintains its own users and passwords, which are separate from the Pi's user accounts. It is not required that your PostgreSQL users match the Pi users. [2]

Reload:

$ sudo /etc/init.d/postgresql reload

Happy hacking!
